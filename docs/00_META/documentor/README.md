# Documentor Domain

This folder serves as the **Documentor** domain for the project. It will contain documentation, guidelines, templates, and any artifacts related to documenting the system.

## Suggested Sub‑Structure
- `overview.md` – High‑level overview of the documentation strategy.
- `architecture.md` – Detailed architecture diagrams and explanations.
- `guidelines.md` – Writing guidelines, style guide, and contribution process.
- `templates/` – Reusable markdown templates for new docs.
- `assets/` – Images, diagrams, and other media used in documentation.

Feel free to add, modify, or delete files here as needed. The total size of this folder must stay under **100 MB**.
